Features

Built for real usage

Not a demo. A working voice pipeline with the pieces you need for reliable phone conversations with Claude.

⏱

Voice Activity Detection

Energy-based VAD with configurable silence threshold and RMS energy floor. Detects when you stop talking, not when you pause.

vad.rs
🚫

Whisper Hallucination Filter

Filters common Whisper artifacts from silence and background noise. No phantom transcriptions hitting Claude.

stt.rs
🗨

Multi-turn Conversations

Session memory per call with configurable timeout. Claude remembers context across turns within the same conversation.

session_timeout_secs
📡

Outbound Call API

POST to /api/call with a phone number and optional message. Trigger calls from n8n, curl, or any automation.

POST /api/call
⚡

Real-time Audio Pipeline

Mu-law codec, streaming WebSocket, direct PCM conversion. No intermediate servers -- Twilio talks straight to Axum.

audio.rs
💬

Configurable Greeting

Custom TTS greeting when a call connects. Set it in config.toml -- default is "Hello, this is Morpheus".

config.toml

Quick start

Up and running in minutes

Clone, configure, deploy. You'll need a Twilio number and API keys for Groq and ElevenLabs.

1

Clone and build

git clone https://github.com/dnacenta/morpheus-line.git cd morpheus-line cargo build --release
2

Configure

mkdir -p ~/.morpheus-line cp config.example.toml ~/.morpheus-line/config.toml cp .env.example ~/.morpheus-line/.env # Edit .env with your API keys # Edit config.toml with your Twilio number
3

Deploy and run

# Copy the binary, set up nginx + systemd # See the full guide in the README sudo cp target/release/morpheus-line /usr/local/bin/ sudo systemctl enable --now morpheus-line
4

Point Twilio to your server

# In Twilio Console, set voice webhook to: POST https://your-server.example.com/twilio/voice
